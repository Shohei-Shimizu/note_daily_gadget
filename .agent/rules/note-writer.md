## Core Principles (執筆原則)
1. **事実と感想の切り分け**: 提供情報は事実として記載し、推測は必ず「〜と感じられる」「〜しやすそう」とトーンを和らげる。
2. **価格情報は出典明記**: 提供されている場合のみ「(情報提供: YYYY年MM月X日現在)」など注記を添える。
3. **ノー挨拶・ノー自分語り**: 「こんにちは」などの挨拶や、自分の体験談・主観的な使用報告は書かない。
4. **「よ」終わりを避ける**: 「〜ですよ」「〜できますよ」などの語尾は禁止。「〜です」「〜ます」「〜ください」で終わらせる。
5. **導入は最短距離で本題へ**: 冗長な前置きを排除し、読者のメリットだけを伝えて記事に入る。
### 禁止事項
- ❌ **ブログ的な比喩表現（例: スパゲッティ状態、相棒、魔法のような）**。
- ❌ **本文内での「商品名: 〇〇」という再記述（見出しにあるため不要）**。
- ❌ **スペック欄に価格・「なし」項目・保証・レビュー評価を記載しない**。
- ❌ **原産国（原産地）を記載しない**。
- ❌ 商品特性を断定する決めつけ表現（例: 「絶対に〜できる」）
- ❌ 200文字超の長段落や箇条書き5項目超の羅列
- ❌ 外部メディアや競合を不当に貶める記述
- ❌ 出典不明の口コミ引用

## Refinement Flow (執筆フロー)
### Phase 1: データ読み込み
- [05_script/search_amazon_products.py](cci:7://file:///Users/shoheishimizu/Knowledge/note-writer/05_script/search_amazon_products.py:0:0-0:0) が出力した JSON データ（`title`, `url`, `price`, `features`）読み込む。
- 必要に応じて、商品名が長すぎる場合は**適切な長さに短縮・最適化**する（型番の削除など）。
### Phase 2: コンテンツ生成（執筆）
- **導入文**: 「導入文の構成」ルールに従って執筆する。
- **商品説明の見出し（H2）**:
    - **見出しの最適化**: JSON内の `title` はあくまで指標である。記事に適した**わかりやすい名称**へとAI判断で書き換える。
    - 形式は `## ① {商品名}` に統一する。
- **商品説明本文**:
    - JSON内の `features` を元にリライトする。
    - **構造遵守**: `H2見出し` の直後、1行空けて `URL` を配置し、その下から本文を書き始める。**「商品名:〜」などの行は入れない。**
    - **1商品2段落以内**: 各段落1〜2文で、要点（効能・使いどころ）を淡々と伝える。
- **あわせて読みたい**: [_metadata.json](cci:7://file:///Users/shoheishimizu/Knowledge/note-writer/02_article/_metadata.json:0:0-0:0) から関連過去記事URLを **5〜10件** 抽出する。
- **最後に**: 総括・行動促進・定型CTA（媒体紹介）の3段落構成で執筆する。
### Phase 3: クリーニング & 自己推敲 (Self-Correction)
- **記事完成後、直ちに以下のチェックリストに基づいて自己レビューを行う**。
    1. **トーンチェック**: 「〜ですよ」語尾や挨拶、**安っぽい比喩**が含まれていないか？
    2. **構成チェック**: `URL` の下に不要な「商品名」行が入っていないか？
    3. **禁止事項チェック**: 価格、保証、レビュー件数などがスペックに含まれていないか？
    4. **リンク実在チェック**: 「あわせて読みたい」のURLは [_metadata.json](cci:7://file:///Users/shoheishimizu/Knowledge/note-writer/02_article/_metadata.json:0:0-0:0) から**コピー＆ペースト**されているか？一文字でも異なるとリンク切れになるため、文字列の完全一致を必ず確認する。
- **修正実行**: もし問題があれば、納品前に**即座に修正（rewrite）**を行う。
- **完了処理**: 全てのチェックをパスした後、[schedule_2026.md](cci:7://file:///Users/shoheishimizu/Knowledge/note-writer/03_schedule/schedule_2026.md:0:0-0:0) の該当タイトルを `~~打ち消し線~~` で囲む。

## 詳細執筆ルール
### 導入文の構成（5文・3ブロック構成・各ブロック間は必ず空行）
読者の「共感」「メリット」「興味」を押さえつつ、**実用的で落ち着いたトーン**で記述する。
**重要: 各ブロック内では改行せず、文章を続けて記述すること。**
**ブロック1（現状・課題）**
1. **第1文（課題の客観視）**: 読者が置かれている状況や一般的な課題を、冷静に指摘する。（例: 「デスクワークにおいて、配線整理は多くの人の課題です。」）
(↓ここに必ず空行を入れる)
**ブロック2（解決策の提示）**
2. **第2文（解決の条件）**: 課題解決のための条件を提示する。
3. **第3文（ベネフィット）**: それによって得られる実務的なメリットを簡潔に伝える。**（第2文から改行せず続ける）**
(↓ここに必ず空行を入れる)
**ブロック3（記事の目的）**
4. **第4文（記事のゴール）**: 「今回は〜を○つ紹介します」と明確に宣言する。
5. **第5文（行動促進）**: 「環境構築の参考にしてください。」等の控えめな表現で締める。**（第4文から改行せず続ける）**
### 本文・スペック
- **スペック（引用スタイル）**: `>**スペック**` の後に `・項目名：内容`。全角中黒使用。最大5〜7項目。
### 最後に
- **3段落構成**: 
  1. 第1段落（総括）: 記事内容の振り返り。
  2. 第2段落（行動促進）: 期待されるアクション・効果。
  3. 第3段落（定型CTA）: 「毎日ガジェット便り」にて、仕事と生活をラクにするガジェット情報を毎日発信している旨と、フォローのお願い。
### あわせて読みたい
- 記事の末尾、`## 最後に` の**後**に配置。
- **各URLの間に必ず一行以上の空行を入れる**（ノートカード表示のため）。
