# Note Article Writer
**説明**: noteで公開する記事を、ユーザーが提示するテーマ・商品リスト・参考情報をもとに共同生成するためのスキル。不正確なURL生成を排除し、実機確認済みの最新情報のみを掲載します。
**このスキルの特徴**: 情報整理→構成設計→本文執筆→補足情報提示→**自己推敲**の5フェーズを明示し、note特有の読みやすさと情報の正確性を徹底的に担保します。
---
## 🛡️ Core Principle: Amazonリサーチフロー (最優先事項)
商品情報の取り扱いについて、以下のプロセスを全てのフェーズで遵守すること。詳細は [.agent/rules/amazon-url-research.md](cci:7://file:///Users/shoheishimizu/Knowledge/note-writer/.agent/rules/amazon-url-research.md:0:0-0:0) を参照。
1. **推測の禁止**: 商品名からURL（`amazon.co.jp/dp/ASIN`等）を文字列操作で勝手に作成しない。
2. **サクラチェッカーによる安全性確認**: 必ず `https://sakura-checker.jp/` またはそのランキングを使用し、サクラ度を確認する。「危険」判定の製品は原則として除外する。
3. **ブラウザによる実機確認**: 必ず `search_web` または `browser_subagent` を使用し、実際にAmazonの商品ページにアクセスする。
4. **生存確認**: 404エラー、在庫切れ（現在取り扱いできません）、モデル違い（型落ち等）を必ずチェックし、問題がある場合はURLを提示しない。
5. **URLのクリーンアップ**: トラッキングパラメータを削除した最小構成（`https://www.amazon.co.jp/dp/[ASIN]`）に整える。
6. **ステータスの明示**: 承認時や執筆時に ✅ `[サクラ除去済+実機確認済]` と添える。

## Core Principles
### 執筆方針
1. **読者の課題から導入する**: 冒頭で読者の状況・課題を描写し、本文に引き込む。
2. **1商品1見出し+2段落以内**: 見出し（H2）は商品名のみ。本文は2段落以内・各段落1〜2文で、要点（効能・使いどころ）を素早く伝える。
3. **事実と感想の切り分け**: ユーザー提供情報は事実として記載し、推測は必ず「〜と感じられる」「〜しやすそう」とトーンを和らげる。
4. **価格情報は出典明記**: 提供されている場合のみ「(情報提供: YYYY年MM月X日現在)」など注記を添える。
5. **note読者向け語調**: 友人におすすめする感覚で丁寧語8割・くだけた表現2割を目指す。
6. **導入は最短距離で本題へ**: 冒頭は2〜3文で課題提示と記事の約束だけを伝え、すぐ本題に入る。
7. **ノー挨拶・ノー自分語り**: 「こんにちは」などの挨拶や、自分の体験談・主観的な使用報告は書かない。
8. **「よ」終わりを避ける**: 「〜ですよ」「〜できますよ」「〜しますよ」などの語尾はAI臭が強いため使用しない。「〜です」「〜ます」「〜ください」で終わらせる。
### 禁止事項
- ❌ **スペック欄に価格を記載しない**。スペック項目では技術仕様のみに限定する
- ❌ **スペック欄に「なし」項目を記載しない**。
- ❌ **保証情報（保証期間・返金ポリシー）をスペックや本文に記載しない**。
- ❌ **レビュー情報（星評価、レビュー件数）をスペックや本文に記載しない**。
- ❌ **原産国（原産地）をスペックや本文に記載しない**。
- ❌ 未提供の価格・在庫・キャンペーン情報をでっちあげない
- ❌ 商品特性を断定する決めつけ表現（例: 「絶対に〜できる」）
- ❌ 200文字超の長段落や箇条書き5項目超の羅列
- ❌ 外部メディアや競合を不当に貶める記述
- ❌ 出典不明の口コミ引用
- ❌ 「こんにちは」「〜してみました」などの挨拶・自分語り・体験談

## 5フェーズ インタラクティブワークフロー
### Phase 0: 受領情報の確認
- テーマ、想定読者、掲載予定日、トーンを確認。
- 目標とする紹介商品数（例：6選なら6つ）を確認し、全体の進捗管理を準備する。
- 既存のURLが提供されている場合も、後述のPhase 3で再リサーチ（生存確認）を行う。
### Phase 1: コンテキスト設計
1. 読者ペルソナを簡潔に仮定。
2. 読者が抱える課題を3つまで抽出。
3. 記事の約束（読むメリット）を1文で提示。
### Phase 2: 構成決定とファイルの下準備（逐次反映フロー）
- 記事タイトル案を3つ提示。承認されたタイトルで、**即座に `02_article/YYYY-MM-DD_タイトル.md` ファイルを作成**する。
- **命名規則**: ファイル名には日付とタイトルを入れ、スペースを `_` で繋ぐ（例：`2026-01-01_持ち運び最強な充電器6選.md`）。
- タイトル・導入文を書き込み、プレースホルダーを用意。
- [02_article/_metadata.json](cci:7://file:///Users/shoheishimizu/Knowledge/note-writer/02_article/_metadata.json) にエントリを追加。
### Phase 3: 商品の選定・確認（重要：リサーチ強化ステップ）
- **リサーチと提案**: テーマに基づき独自リサーチ。
    - **サクラチェッカー起点**: 可能であれば `https://sakura-checker.jp/ranking/` のカテゴリランキング上位から選定する。
    - **レビュー重視**: 星の数だけでなく、特に日本語レビューの質と「サクラ無し」判定を最優先する。
    - 前述の「Amazonリサーチフロー」を適用し、実機確認済みのクリーンなURLを確保する。
- **提案内容**: 各商品の「名称」「確定URL」「選定理由」「確認状況（✅）」を提示。
- **ユーザー承認**: ユーザーの承認を得て掲載情報を確定。承認された商品から順次（執筆ルールに基づき）執筆を開始。
### Phase 4: 本文完成
- 承認された商品情報に基づき本記事を執筆。
- 全商品が埋まったら、「最後に」「あわせて読みたい」を追加。
- (※この段階ではまだ完了しない)
### Phase 5: Self-Correction (品質担保・自己推敲)
- **記事完成後、直ちに以下のチェックリストに基づいて自己レビューを行う**。
    1. **トーンチェック**: 「〜ですよ」語尾や挨拶が含まれていないか？
    2. **URLチェック**: 全てのURLが実機確認済みで、トラッキングIDが削除されているか？
    3. **禁止事項チェック**: 価格、保証、レビュー件数などがスペックに含まれていないか？
    4. **サクラチェック**: 選定した製品が本当にサクラチェッカー合格基準を満たしているか？
    5. **導入文フォーマット**: **「1文目＋空行／2・3文目＋空行／4・5文目」**の3ブロック構成になっているか？
- **修正実行**: もし問題があれば、ユーザーに納品する前に**即座に修正（rewrite）**を行う。
- **完了処理**: 全てのチェックをパスした後、[04_schedule/schedule_2025_2026.md](file:///Users/shoheishimizu/Knowledge/note-writer/04_schedule/schedule_2025_2026.md) の該当タイトルを `~~打ち消し線~~` で囲む。

## 詳細執筆ルール
### 導入文の構成（5文・3ブロック構成・各ブロック間は必ず空行）
読みやすさを重視し、以下の3ブロックに分けて記述する。**HTMLの`<br>`等ではなく、Markdownの空行で視覚的に区切ること**。
**ブロック1（共感）**
1. **第1文（課題・共感）**: 疑問形で読者の状況に共感（例: 〜ませんか）
(↓ここに必ず空行を入れる)
**ブロック2（深掘り）**
2. **第2文（問題の具体化）**: 具体的なメリット提示。
3. **第3文（解決策提示）**: 条件形「〜なら」でベネフィットを伝える。
(↓ここに必ず空行を入れる)
**ブロック3（約束と行動）**
4. **第4文（記事の約束）**: 「今回は、〜を○つ紹介します」と明示。
5. **第5文（行動促進）**: 「気になった商品があればぜひチェックしてみてください」で締める。
- **改善例**: `❌ 年末年始は写真が増えます〜 ⭕ 年末年始は写真が増えると思います。撮った写真をその場ですぐにプリントできれば〜`（※連続する「ます」を避ける）
### 本文・スペック
- **商品H2**: 丸数字（①②③...）＋半角スペース＋商品名。
- **URL配置**: H2の直後、1行空けてURLを剥き出し（クリーン版）で配置。その後に説明文。
- **スペック（引用スタイル）**: `>**スペック**` の後に `・項目名：内容`。全角中黒使用。最大5〜7項目。
### あわせて読みたい
- 記事の末尾に配置。`_metadata.json` からカテゴリ・タグ・テーマが近い過去記事を5〜10件抽出。
- **各URLの間に必ず一行以上の空行を入れる**（ノートカード表示のため）。
### 最後に
- **3段落構成**: 
  1. 第1段落（総括）: 記事内容の振り返り。
  2. 第2段落（行動促進）: 期待されるアクション・効果。
  3. 第3段落（定型CTA）: 「毎日ガジェット便り」の媒体紹介とフォロー促進。
