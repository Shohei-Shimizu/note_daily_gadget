# Generate Article Thumbnail Prompt

### 目的
note記事のテーマと内容から、サムネイル画像生成用のプロンプト（テキスト）を作成し、Pencil（.pen）ファイル上にフレーム＋プロンプトを一体で配置する。画像生成は任意の外部ツール（DALL-E、Midjourney、Stability AI 等）で行う前提。

### 入力
- 記事のテーマ・タイトル
- 記事内で紹介している商品の情報（商品名・外観的特徴）
- 任意で `style_hint`（色味・季節感など）

---

### 出力先: Pencil（.pen）ファイル

**月ごとに1つの `.pen` ファイル**を使い、記事ごとのサムネイルブロックを**横に並べて追記**していく。

- **ファイルパス**: `04_thumbnail/YYYY-MM/pencil-new.pen`
- **月が変わったら**: 新しい月のディレクトリ（例: `04_thumbnail/2026-04/`）を作成し、その中に新規 `pencil-new.pen` を作成する

### Pencil上のレイアウト構成（1記事分のブロック）

各記事のサムネイルブロックは以下の構造で、既存ブロックの**右側**に配置する（`positionDirection: "right"`, `positionPadding: 100`）。

```
[記事タイトル（ヘッダー）]
[区切り線]
[1200×628 フレーム（画像貼り付け用・白背景・枠線付き）]
[「↑ ここに画像を配置（1200×628）」ラベル]
[区切り線]
[Thumbnail Prompts ラベル]
[Scene 1 カード: タイトル / Prompt / Japanese / Focus]
[Scene 2 カード: タイトル / Prompt / Japanese / Focus]
[Scene 3 カード: タイトル / Prompt / Japanese / Focus]
```

### Pencil操作手順

1. `04_thumbnail/YYYY-MM/pencil-new.pen` を `open_document` で開く
   - ファイルが存在しない（月初など）場合は `new` で作成後、そのディレクトリに保存される
2. `find_empty_space_on_canvas` で既存コンテンツの右側に空きスペースを確認
3. `batch_design` で以下を作成（最大25 ops/call、複数callに分割可）:
   - 外枠フレーム（`placeholder: true`、`layout: "vertical"`, `width: 1300`）
   - ヘッダーテキスト（日付＋記事タイトル）
   - 区切り線
   - サムネイルフレーム（`width: 1200`, `height: 628`, **`layout: "none"`, `clip: true`**, 白背景＋枠線）※画像貼り付け用の #frame
   - ラベルテキスト
   - 区切り線
   - プロンプトセクション見出し
   - Scene 1〜3 カード（白背景・角丸8px）
4. `placeholder: false` に更新
5. `get_screenshot` で確認

### スタイル定義（Pencilノード）

| 要素 | スタイル |
|---|---|
| 外枠フレーム | `fill: "#F5F5F5"`, `gap: 32`, `padding: 32` |
| ヘッダー | `fontSize: 24`, `fontWeight: "bold"`, `fill: "#1A1A1A"` |
| 区切り線 | `rectangle`, `height: 1`, `fill: "#CCCCCC"` |
| サムネイルフレーム | `fill: "#FFFFFF"`, `stroke: inside 1px #E0E0E0`, **`layout: "none"`, `clip: true`** |
| ラベル | `fontSize: 14`, `fill: "#888888"`, `textAlign: "center"` |
| Scene カード | `fill: "#FFFFFF"`, `cornerRadius: 8`, `padding: 16`, `gap: 8` |
| Scene タイトル | `fontSize: 16`, `fontWeight: "bold"`, `fill: "#333333"` |
| Prompt / Japanese | `fontSize: 13`, `fill: "#555555"`, `lineHeight: 1.6` |
| Focus | `fontSize: 13`, `fontWeight: "bold"`, `fill: "#0066CC"` |

### 共通スタイル要件（プロンプト内容に必ず反映）

- **スタイル**: フォトリアルで高品質なライフスタイル写真（実写に見えるクオリティ）
- **シーン設定（In-Use Scenario）**:
  - 単にモノが置いてあるだけでなく、**「実際に使われている」状態を描く**
  - 例: ケーブルが繋がっている、スマホ画面が点灯している、手帳が開かれている
  - メインのオブジェクトはテーブル、デスク、棚などの上に「置かれて」いる
- **ライティング**: 「窓際の自然光」。片側から柔らかいサイドライト。ハイキー（明るめ）トーン
- **カラーパレット**: モダン＆ミニマル。「ホワイト・ナチュラルウッド・ブラック（アクセント）」
- **構図**:
  - 背景ボケ: 奥行きのある室内が背景。窓枠や観葉植物が美しくボケている
  - プロップ: ノートPC、スマホ、コーヒーマグ、観葉植物などで生活感を演出
  - トリミング: 正方形→16:9トリミングを前提。メイン被写体は中央の水平帯（縦56%）に収める
- **避けるもの**: テキスト全般、実在するロゴ、宙に浮いている物体、非現実的なネオン発光、顔のアップ

### 商品の外観をプロンプトに反映する

- リサーチデータ（JSON）の商品情報から、代表的な商品の**外観的特徴（形状、色、素材、ボタン配置など）**を把握する
- 「汎用的なデバイス」ではなく、**「その商品そのもの」に見えるレベル**まで具体的に描写する
- 例: 「角が丸い四角形」「前面に大きな丸いスピーカー穴」など
- ロゴマークは含めない（崩れるため）。形状と雰囲気を優先

### プロンプト設計の手順
1. 記事テーマから主題となるアイテム・シーン・雰囲気を抽出
2. 共通スタイル要件を基盤に、主題を表現する3つ以上のシーンバリエーションを設計
3. 各シーンで焦点を変える（クローズアップ、異なる室内背景、光の時間帯）ことで差別化
4. Pencilファイルにフレーム＋プロンプトを配置

### 画像の書き出し（ユーザーが画像貼り付け後）
- Pencilの `export_nodes` でサムネイルフレームをPNG書き出し
- 保存先: `04_thumbnail/YYYY-MM/`
- ファイル名: `YYYY-MM-DD_テーマ_概要.png`
