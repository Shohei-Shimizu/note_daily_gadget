# Note Article Writer (PA-API Integrated)

## **説明**: Amazon PA-API から取得した商品検索結果（JSON）とテーマを元に、note 公開用の完成記事形式に整形・執筆するスキル。

## 🛡️ Core Principle: URL とデータの取り扱い

**AI は、提供された API データ内の URL（アフィリエイトリンク）をそのまま使用する。**
記事作成時は、必ず [05_script/search_amazon_products.py](05_script/search_amazon_products.py) の出力結果に基づくこと。

---

## ブランドプロフィール: 毎日ガジェット通信

-   **目的**: 「毎日ガジェット通信」は、"仕事と生活をラクにするガジェット"を紹介し、読者の QOL 向上と収益化を両立するメディアである。
-   **キャラクター**: 現場目線を大事にする編集部の声。
-   **想定読者**: 20〜40 代のナレッジワーカー／クリエイター
-   **トーン**: **「丁寧で実用的なマガジンスタイル」**。
    -   基本は「です・ます調（敬体）」。
    -   ブログ的な馴れ馴れしさ（「〜ですよね？」）は排除するが、新聞ほど硬くせず、読者に寄り添う柔らかさを意識する。
    -   語尾は「〜してください」と断定せず、「〜してみてください」と提案する形を好む。
-   **詳細メモ**: [.docs/note_account_design.md](.docs/note_account_design.md)

## 🎯 想定ユースケース

-   **商品リサーチと URL 発行は [05_script/search_amazon_products.py](05_script/search_amazon_products.py) が行い、AI は執筆・整形（清書）に専念する。**

## Core Principles (執筆原則)

1. **事実と感想の切り分け**: 提供情報は事実として記載し、推測は必ず「〜と感じられる」「〜しやすそう」とトーンを和らげる。
2. **価格情報は出典明記**: 提供されている場合のみ「(情報提供: YYYY 年 MM 月 X 日現在)」など注記を添える。
3. **ノー挨拶・ノー自分語り**: 「こんにちは」などの挨拶や、自分の体験談・主観的な使用報告は書かない。
4. **「よ」終わりを避ける**: 「〜ですよ」「〜できますよ」などの語尾は禁止。「〜です」「〜ます」「〜ください」で終わらせる。
5. **導入は最短距離で本題へ**: 冗長な前置きを排除し、読者のメリットだけを伝えて記事に入る。

### 禁止事項

-   ❌ **ブログ的な比喩表現（例: スパゲッティ状態、相棒、魔法のような）**。
-   ❌ **本文内での「商品名: 〇〇」という再記述（見出しにあるため不要）**。
-   ❌ **スペック欄に価格・「なし」項目・保証・レビュー評価を記載しない**。
-   ❌ **原産国（原産地）を記載しない**。
-   ❌ 商品特性を断定する決めつけ表現（例: 「絶対に〜できる」）
-   ❌ 200 文字超の長段落や箇条書き 5 項目超の羅列
-   ❌ 外部メディアや競合を不当に貶める記述
-   ❌ 出典不明の口コミ引用

## Refinement Flow (執筆フロー)

### Phase 1: データ読み込み

-   Codexが `.agent/rules/note-researcher.md` の基準で選定・保存した **JSONデータ（`title`, `url`, `price`, `features` 等が含まれているもの）** を読み込む。
-   必要に応じて、商品名が長すぎる場合は**適切な長さに短縮・最適化**する（型番の削除など）。

### 記事フォーマットテンプレート

全ての記事は以下の構造に厳密に従うこと:

**テンプレート構造**:
- タイトル (H1)
- 導入文（**2つの島**: 島1=第1〜2文、空行、島2=第3文）
- 商品セクション（見出しH2 → 空行 → URL → 空行 → **本文1段落** → 空行 → スペック）
- 最後に (H2 + 2段落構成: 第1段落=総括と行動促進、第2段落=定型案内とCTA）
- あわせて読みたい (H2)

**段落ルール（改行の禁止）**:
- **導入文**: 
  - 島1: 第1文と第2文を改行なしで繋げる
  - 空行を1つ入れる
  - 島2: 第3文のみ（改行不要）
- **商品説明**: 複数文を改行せず、1つの段落にまとめる（3〜4文・150〜200文字程度）
- **最後に**: 
  - 島1: 総括と行動促進を改行なしで繋げる
  - 空行を1つ入れる
  - 島2: 定型案内とCTA（スキとフォロー）を改行なしで繋げる

**空行ルール（島を作る）**:
- タイトルと導入文の間: 1行空行
- 導入文の島1と島2の間: 1行空行
- 導入文と最初の商品セクションの間: 1行空行
- 見出し (H2) と URL の間: 1行空行
- URL と本文の間: 1行空行
- 本文とスペックの間: 1行空行
- 商品セクション間: 1行空行（H2の前）
- 最後の商品とまとめの間: 1行空行
- まとめの島1と島2の間: 1行空行
- まとめとあわせて読みたいの間: 1行空行
- あわせて読みたいの各URL間: 1行空行

**禁止事項**:
- **URL の直下に「商品名: 〜」などの再記述は絶対に禁止**
- 導入文・商品説明・まとめの各島内での改行は禁止（段落内は改行なし）

**参考**: 既存記事 [02_article/2026-03/2026-03-01_アナログに見えて実は高性能。大人の「ハイブリッド時計」_6選.md](02_article/2026-03/2026-03-01_アナログに見えて実は高性能。大人の「ハイブリッド時計」_6選.md) のフォーマットを完全に模倣すること。

### Phase 2: コンテンツ生成（執筆）

-   **導入文**: 「導入文の構成」ルールに従って執筆する。
-   **商品説明の見出し（H2）**:
    -   **見出しの最適化**: JSON 内の `title` はあくまで指標である。記事に適した**わかりやすい名称**へと AI 判断で書き換える。
    -   形式は `## ① {商品名}` に統一する。
-   **商品説明本文**:
    -   JSON 内の `features` を元にリライトする。
    -   **構造遵守**: `H2見出し` の直後、1 行空けて `URL` を配置し、その下から本文を書き始める。**「商品名:〜」や「💡注目ポイント」などの余分な行は絶対に入れない。**
    -   **完全改行なし（1段落にまとめる）**: 複数文になっても**絶対に途中で改行せず、1つの段落（3〜4文・150〜200文字程度）にまとめる**こと。段落内の改行は厳禁。要点（効能・使いどころ）を淡々と伝える。
-   **あわせて読みたい**: [\_metadata.json](02_article/_metadata.json) から関連過去記事 URL を **5〜10 件** 抽出する。
-   **最後に**: 総括＋行動促進の段落と、定型媒体紹介＋CTAの2段落構成で執筆する。アフィリエイト開示文は不要。

### metadata.json 更新ルール（必須手順）

記事（Markdown）の保存が完了したら、**必ず続いて `02_article/{YYYY-MM}/_metadata.json` ファイルを開き、新規記事のデータを直接追記すること。**

**必須ルール**:
- `url`: 必ず空文字列 `""` を設定する（アフィリエイトURLは記事本文内に配置済みのため。`urls: []` などは不可）。
- `tags`: JSON配列ではなく、**スペース区切りのプレーンテキスト**で設定する。（Phase 2.5 タグ選定ルール参照）
- `category`: 対象商品に適したカテゴリを既存のメタデータを参考に**必ず**設定する（生成漏れ厳禁）。
- 既存の JSON 構造（`articles` 配列）を壊さないように注意すること。

### Phase 3: クリーニング & 自己推敲 (Self-Correction)

-   **記事完成後、直ちに以下のチェックリストに基づいて自己レビューを行う**。
    1. **トーンチェック**: 「〜ですよ」語尾や挨拶、**安っぽい比喩**が含まれていないか？
    2. **構成チェック**: `URL` の直下に不要な「💡注目ポイント」や「商品名」行が入っていないか？
    3. **禁止事項チェック**: 価格、保証、レビュー件数などが入っていないか？アフィリエイト明記文が入っていないか？
    4. **リンク実在チェック**: 「あわせて読みたい」の URL は文字列の完全一致で記載されているか？
-   **修正実行**: もし問題があれば、納品前に**即座に修正（rewrite）**を行う。

## 詳細執筆ルール

### 導入文の構成（短潔・余白重視）

**目的: 読者を最短距離で本題（商品リスト）へ誘導する。全体の長さは3〜4文程度、200文字以内に収めること。**

1. **第 1 文（背景と課題）**: 読者の潜在的な悩みやニーズを一言で代弁する。（例: 「〜は意外と面倒なものです。」「〜に悩む方は少なくありません。」）
2. **第 2 文（解決策）**: 記事のテーマがその悩みをどう解決するかを簡潔に提示する。（例: 「そんなときは〜が役立ちます。」）
3. **第 3 文（提案・結び）**: 今回紹介する商品数と、読者への提案を行う。（例: 「今回は〜を厳選しました。ぜひお気に入りの一台を見つけてみてください。」）※原則この文で導入を終える。
4. **（オプション）**: 文脈上どうしても必要な場合のみ、間に1文追加してもよいが、原則3文構成とする。

### 太字マーカーの装飾ルール

- **選定基準などの太字は1ブロックにつなげる**: note の表示崩れ回避のため、`**「A」、「B」、「C」**` の形式を使う。
- **マーカー前後のスペース**: 文中の太字マーカーの前後に必ず半角スペースを入れる（例: `今回は **「A」、「B」、「C」** の3点で選びました。`）。
- ❌ **禁止**: `今回は**「A」**、**「B」**、**「C」**の3点...` といった細分化された太字の使用は禁止。

### 商品説明本文の文量とリズムルール

**3〜4文・150〜200文字程度**に収める。改行は入れず、複数文を続けて1段落にまとめる。

**構成の目安**:
- 1文目：どんな商品か・一見した際の特徴
- 2文目：主要機能・実用的なメリット
- 3文目（必要に応じて4文目）：バッテリーなどの基本性能・安心感

**スペックの箇条書きと同じ内容（重量の正確な数値など）は本文に書かない。** ただし「軽量である」「長時間バッテリー」といった表現はOK。

**文体のリズム（自然な日本語）**:
- **同じ語尾を連続させない。** 2文以上「〜ます」が続く場合は、必ず別の語尾に変える。
- 語尾バリエーション: 体言止め（〜な設計。）、〜はず（〜使いやすいはず。）、過去形（〜選びました。）、〜です。
- ❌ `〜違います。〜減らせます。〜選んでいます。`（ます3連続）
- ✅ `〜違います。〜減らせるはず。〜3点で選びました。`（リズムが生まれる）

### 編集部の人格を出す表現ルール（調べ手スタンス）

毎日ガジェット通信は**「徹底的に調べてまとめた編集部」**のスタンスを取る。「実際に使ってみました」「買ってよかった」といった実使用の断言は禁止だが、以下の切り口を商品説明文に織り込むことで自然な説得力を出す。

**切り口（1商品に一つ程度）**:
- **選定理由の開示**: 「〇〇を重視して候補に入れた一台」「〜向けで調べていて気になった点」
- **比較・消去法**: 「〇〇と迷う場面が多い中で、これを選ぶ理由は〜」「〜が不要ならこれで十分」

**禁止事項**:
- ❌ レビュー由来の事実（件数、星、傾向）への言及
- ❌ 実使用の感想（「私は〜」「実際に使って〜」）

### 本文・スペックの記述

-   **スペック（引用スタイル）**: `>**スペック**` の後に `・項目名：内容`。全角中黒使用。最大 5〜7 項目。

### 最後に

-   **2 段落構成（改行位置を固定）**:
    1. **第1段落（総括と行動促進）**: 記事内容の振り返りと、読者への提案（「ぜひ、ご自身のファッションやライフスタイルに合った最適な1台を見つけてみてください。」など）を1段落で書く。
    2. **第2段落（定型案内とCTA）**: 「毎日ガジェット通信では、仕事と生活をラクにするガジェット情報を発信しています。気に入っていただけたら、ぜひスキとフォローをお願いします！」という定型文を記述する。
-   **禁止**: CTAを独立した1行に分離しない。アフィリエイト明記も不要のため書かない。

### あわせて読みたい

-   記事の末尾、`## 最後に` の**後**に配置。
-   **各 URL の間に必ず一行以上の空行を入れる**（ノートカード表示のため）。

### Phase 2.5: タグ選定ルール (metadata.json追加用)

記事完成時に `_metadata.json` 向けに準備するタグ（`tags`）は、以下の戦略に従い、**多すぎるタグ・類似タグの重複を避けて** 厳選すること。

#### タグの選び方（最大5〜8個に厳選）
| 層 | 選び方のルール | 候補タグ例（※重複注意） |
|---|---|---|
| **公式企画枠 (1〜2個)** | note公式の流入を獲得 | `#買ってよかったもの` または `#おすすめガジェット` |
| **キーワード枠 (1〜2個)** | 検索・回遊用 | `#QOL` または `#ベストバイ` |
| **カテゴリ枠 (1〜2個)** | ターゲット層を直撃 | `#デスク環境` `#生産性` `#生活改善` |
| **記事固有タグ (1〜2個)** | 商品カテゴリ等 | （例：`#Anker` `#モバイルバッテリー` など） |

#### ⚠️ タグ選定の絶対ルール（類似タグの併用禁止）
- **タグ総数**: **最大5〜8個程度** に厳格に収める（10個以上は不可）。
- **意味の重複・表記揺れの同時使用禁止**:
  - ❌ `#買ってよかった` と `#買ってよかったもの` の同時使用
  - ❌ `#QOL` と `#QOL向上` と `#QOLあげてみた` の同時使用
  - ❌ `#暮らし` と `#丁寧な暮らし` と `#一人暮らし` の過剰な同時使用
  - （※文脈に最も合う**代表的な1つに絞って**出力すること）
